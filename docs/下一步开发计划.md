# 下一步开发计划

## 当前状态

已完成：

1. Python/Conda 环境。
2. 字段标签体系。
3. 训练样本集。
4. 贝叶斯字段分类。
5. 本地规则分级。
6. CSV/JSON 报告导出。
7. 分类模型评估报告。

## 建议继续顺序

### 第一步：扩充样本

当前样本是为了跑通原型准备的示例数据。后续应使用真实业务字段扩充训练样本。

建议每个标签至少准备 20 条样本，优先覆盖：

- 字段名简写
- 中英文混合字段名
- 字段注释为空
- 样例值为空
- 同一字段在不同业务场景下的含义差异

### 第二步：优化标签体系

当前标签体系适合原型，但真实项目需要按单位制度或行业规范细化。

建议补充：

- 数据大类
- 数据子类
- 个人信息类别
- 敏感个人信息类别
- 重要数据识别线索
- 默认安全等级
- 推荐保护措施

### 第三步：增加人工复核表

分类分级结果不能完全依赖模型，需要增加人工确认字段。

建议报告中增加：

- 人工确认标签
- 人工确认等级
- 修改原因
- 复核人
- 复核时间

### 第四步：接入大模型或本地模型

没有 API key 时继续使用本地规则分级。

后续可选：

- 接通用大模型 API
- 接 Ollama 本地模型
- 保留本地规则作为兜底

### 第五步：完善界面

当前已完成 Gradio 本地页面。

后续建议继续补：

- 人工复核列
- 复核结果保存
- 标签体系在线查看
- 评估报告页面化
- 批量历史记录查看

如果后续要做成正式工具，再考虑 FastAPI + 前端。
