# 更新文献推荐

## 1. 推荐结论

原先参考的 Sherlock 论文是 2019 年的经典字段语义类型识别文献，但用于当前项目汇报时确实偏旧。

更建议当前项目引用以下 2024 年之后的文献：

1. 主推：RACOON，2024
2. 辅助：Column Vocabulary Association，2024
3. 进阶参考：LLM Agent-Based Semantic Table Annotation，2025
4. 前沿参考：ZTab，2026

其中，最适合放进当前项目方案的是：

```text
RACOON: An LLM-based Framework for Retrieval-Augmented Column Type Annotation with a Knowledge Graph
```

原因是它直接研究 Column Type Annotation，也就是表格列/字段的语义类型标注，和本项目“对数据字段打标”高度一致。

## 2. 主推文献

### RACOON: An LLM-based Framework for Retrieval-Augmented Column Type Annotation with a Knowledge Graph

- 时间：2024 年 9 月提交，2024 年 11 月修订
- 作者：Lindsey Linxi Wei, Guorui Xiao, Magdalena Balazinska
- 方向：数据库、人工智能
- 链接：https://arxiv.org/abs/2409.14556
- DOI：https://doi.org/10.48550/arXiv.2409.14556

### 核心内容

该论文研究 Column Type Annotation，即给表格列打语义类型标签。

论文提出 RACOON 框架，将大语言模型和知识图谱结合，通过检索增强方式补充上下文信息，从而提升列类型标注效果。

它的核心思想是：

```text
表格列信息
    |
    v
检索知识图谱相关上下文
    |
    v
LLM 进行列类型标注
```

### 和本项目的关系

本项目当前做的是：

```text
字段名 / 字段注释 / 字段类型 / 样例摘要
    |
    v
字段分类标签
```

RACOON 做的是：

```text
表格列
    |
    v
语义类型标签
```

两者问题形式非常接近。

本项目可以这样引用它：

> 在字段分类打标方面，可参考 Column Type Annotation 相关研究。RACOON 通过 LLM 与知识图谱检索增强结合，提高表格列语义类型标注效果。当前项目原型阶段采用贝叶斯分类器和规则特征实现轻量可解释 baseline，后续可参考 RACOON 思路引入知识库/RAG/LLM 作为低置信度字段的增强判断模块。

## 3. 辅助文献

### Column Vocabulary Association (CVA): semantic interpretation of dataless tables

- 时间：2024 年 9 月
- 作者：Margherita Martorana, Xueli Pan, Benno Kruit, Tobias Kuhn, Jacco van Ossenbruggen
- 链接：https://arxiv.org/abs/2409.13709
- DOI：https://doi.org/10.48550/arXiv.2409.13709

### 核心内容

这篇论文研究不依赖底层真实数据，仅使用元数据进行列语义解释。

它特别适合本项目，因为数据安全场景下通常不能随意把真实数据样例交给模型。

也就是说，本项目可以采用：

```text
字段名
字段注释
数据类型
业务场景
脱敏样例
```

而不是直接使用真实敏感数据。

### 对本项目的启发

这篇文献可以支撑我们当前设计：

```text
不直接上传真实数据
优先使用字段元数据和脱敏摘要进行分类分级
```

同时，它也提到传统方法在某些场景可能超过 LLM，这可以支撑我们“没有 LLM 也能实现基础版本”的技术判断。

## 4. 进阶参考文献

### An LLM Agent-Based Complex Semantic Table Annotation Approach

- 时间：2025 年 8 月
- 作者：Yilin Geng 等
- 链接：https://arxiv.org/abs/2508.12868
- DOI：https://doi.org/10.48550/arXiv.2508.12868

### 核心内容

这篇论文研究复杂表格语义标注，包含：

- Column Type Annotation，列类型标注
- Cell Entity Annotation，单元格实体标注

它使用 LLM Agent 和外部工具，根据表格特征动态选择标注策略。

### 对本项目的启发

这篇更适合后续版本参考：

```text
字段复杂、上下文不足、存在缩写或歧义时
    |
    v
调用 LLM Agent / 工具链辅助判断
```

当前项目还不需要做到 Agent 级别，但可以把它作为未来增强方向。

## 5. 前沿参考文献

### ZTab: Domain-based Zero-shot Annotation for Table Columns

- 时间：2026 年 3 月
- 作者：Ehsan Hoseinzade, Ke Wang
- 链接：https://arxiv.org/abs/2603.11436

### 核心内容

这篇研究零样本列语义类型检测，强调在缺少用户标注数据、存在隐私限制时，如何进行表格字段标注。

### 对本项目的启发

这篇可以支撑后续方向：

```text
当训练样本不足时
    |
    v
通过领域配置、伪表生成、零样本方法提升字段分类能力
```

但它比较新，建议作为前沿参考，不作为当前项目主文献。

## 6. 推荐在项目材料中的写法

建议把文献支撑写成：

```text
字段分类打标属于表格列语义类型标注问题。近年来，Column Type Annotation 和 Semantic Table Annotation 研究逐渐引入 LLM、RAG 和知识图谱等技术。RACOON（2024）提出利用知识图谱增强 LLM 进行列类型标注；CVA（2024）进一步讨论了仅基于元数据进行列语义解释的可行性。结合数据安全场景中真实数据不宜外发的特点，本项目原型阶段采用贝叶斯分类器、字段关键词和正则规则构建轻量可解释的字段打标 baseline，并预留 LLM/RAG 作为低置信度字段和复杂业务场景的增强模块。
```

## 7. 如何替换原 Sherlock 文献

原 Sherlock 可以保留为经典背景，但不建议作为唯一参考。

更推荐：

```text
经典背景：Sherlock，2019
当前主文献：RACOON，2024
隐私/元数据支撑：CVA，2024
未来增强方向：LLM Agent STA，2025
```
