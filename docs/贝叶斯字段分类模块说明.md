# 贝叶斯字段分类模块说明

## 1. 模块定位

贝叶斯模块只负责字段分类，不负责最终数据分级。

它的输入是字段元数据：

- 表名
- 表注释
- 字段名
- 字段注释
- 数据类型
- 少量脱敏样本或样本格式
- 是否可空
- 是否主键
- 是否唯一

输出是字段所属的数据类别、置信度、候选类别和解释证据。

## 2. 核心代码结构

- `src/data_classification_tool/regex_rules.py`：正则规则和样本画像
- `src/data_classification_tool/feature_extractor.py`：字段特征抽取
- `src/data_classification_tool/bayes_classifier.py`：贝叶斯分类器
- `scripts/train.py`：训练模型
- `scripts/predict.py`：预测字段类别
- `scripts/evaluate.py`：评估模型
- `data/sample_train_data.json`：训练数据示例
- `data/sample_predict_data.json`：预测数据示例

## 3. 特征构造

`FieldFeatureExtractor` 会生成适合贝叶斯模型的文本特征。

它会处理：

1. 下划线字段拆分，例如 `user_phone_number` -> `user phone number`
2. 驼峰字段拆分，例如 `userPhoneNumber` -> `user phone number`
3. 中文字段名和注释分词，优先使用 `jieba`
4. 数据类型特征，例如 `varchar`、`decimal`、`date`、`text`
5. 正则特征，例如手机号、邮箱、身份证、银行卡、IP、URL、日期、金额、统一社会信用代码
6. 样本统计特征，例如平均长度、固定长度、纯数字比例、中文比例、高唯一率
7. 强特征加权，例如身份证、手机号、邮箱等特征会重复加入特征文本

注意：原始样本值不会直接进入 `feature_text`。

## 4. 训练数据格式

示例：

```json
{
  "table_name": "user_auth",
  "table_comment": "用户实名认证表",
  "column_name": "id_card_no",
  "column_comment": "身份证号码",
  "data_type": "varchar",
  "sample_values": ["110101********1234", "320102********5678"],
  "nullable": false,
  "is_primary_key": false,
  "is_unique": true,
  "label": "个人身份信息"
}
```

## 5. 运行示例

训练：

```powershell
& "D:\conda\envs\data-classification\python.exe" D:\classification\data-classification-tool\scripts\train.py
```

预测：

```powershell
& "D:\conda\envs\data-classification\python.exe" D:\classification\data-classification-tool\scripts\predict.py
```

评估：

```powershell
& "D:\conda\envs\data-classification\python.exe" D:\classification\data-classification-tool\scripts\evaluate.py
```

## 6. 预测输出

输出包含：

- `predicted_category`
- `confidence`
- `top_candidates`
- `need_review`
- `evidence_features`
- `matched_patterns`
- `sample_profile`
- `feature_text`

其中 `feature_text` 便于调试，不建议直接展示给最终业务用户。

## 7. 与 LLM 分级模块衔接

`DataFieldBayesClassifier.to_llm_grading_input()` 会生成后续 LLM 分级所需输入：

```json
{
  "field_name": "id_card_no",
  "field_comment": "身份证号码",
  "table_name": "user_auth",
  "table_comment": "用户实名认证表",
  "data_type": "varchar",
  "bayes_category": "个人身份信息",
  "bayes_confidence": 0.94,
  "evidence_features": ["身份证正则匹配", "用户实名认证表"],
  "sample_profile": {
    "sample_count": 2,
    "avg_length": 18,
    "fixed_length": true
  },
  "need_review": false
}
```

该输入不包含真实原始样本值，只包含样本画像和证据特征。
