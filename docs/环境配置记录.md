# 环境配置记录

## 结论

本项目可以用 Python 实现，而且第一版非常适合用 Python。

本机原有 `D:\anaconda` 存在，但缺少 `D:\anaconda\python.exe`，导致 `D:\anaconda\Scripts\conda.exe` 无法正常运行。因此没有继续修补旧安装，而是在 D 盘新装了独立 Miniconda。

严格来说，这次安装的是 Miniconda，不是完整版 Anaconda。这样更适合多项目开发：conda 本体更轻，项目依赖按需安装，每个项目单独创建环境。

## 安装结果

- Miniconda 安装目录：`D:\Miniconda3`
- conda 环境目录：`D:\conda\envs`
- conda 包缓存目录：`D:\conda\pkgs`
- 本项目环境名：`data-classification`
- 本项目环境路径：`D:\conda\envs\data-classification`
- 项目目录：`D:\classification\data-classification-tool`

## 为什么这样配置

1. 新的 conda 本体在 D 盘，不影响旧的 `D:\anaconda`。
2. 环境目录在 D 盘，以后不同项目可以分别创建不同环境。
3. 包缓存目录在 D 盘，后续下载的 conda 包不会默认占用 C 盘。
4. 没有自动加入系统 PATH，避免影响其他 Python/软件。
5. 没有把 Miniconda 注册成系统默认 Python，避免污染系统环境。

## 后续使用命令

推荐直接使用 `conda run`，不依赖 PowerShell 是否初始化：

```powershell
& "D:\Miniconda3\Scripts\conda.exe" run -n data-classification python D:\classification\data-classification-tool\scripts\smoke_test.py
```

如果希望在当前 PowerShell 中激活环境：

```powershell
(& "D:\Miniconda3\Scripts\conda.exe" "shell.powershell" "hook") | Out-String | Invoke-Expression
conda activate data-classification
cd D:\classification\data-classification-tool
```

也可以使用项目辅助脚本：

```powershell
. D:\classification\data-classification-tool\scripts\activate_project.ps1
```

注意前面的点号 `.` 表示在当前 PowerShell 会话中执行脚本，这样激活状态才能保留。

## 给以后项目创建新环境

以后有新项目时，建议继续放 D 盘，例如：

```powershell
& "D:\Miniconda3\Scripts\conda.exe" create -n another-project python=3.11 pandas scikit-learn -y
```

新环境会优先创建到：

```text
D:\conda\envs\another-project
```

包缓存会放到：

```text
D:\conda\pkgs
```

## 本项目环境复现

本项目环境声明文件：

```text
D:\classification\data-classification-tool\environment.yml
```

如果以后需要重建环境：

```powershell
& "D:\Miniconda3\Scripts\conda.exe" env create -f D:\classification\data-classification-tool\environment.yml
```
