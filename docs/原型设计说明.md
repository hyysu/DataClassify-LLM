# 数据分类分级工具原型设计说明

## 1. 当前目标

当前版本先实现一个不用大模型 API key 也能运行的最小原型：

1. 读取字段元数据 CSV。
2. 使用贝叶斯模型进行字段分类打标。
3. 使用本地规则进行数据安全分级。
4. 输出 CSV 和 JSON 报告。

## 2. 输入字段

输入 CSV 至少包含以下列：

- `table_name`：表名
- `column_name`：字段名
- `column_comment`：字段注释
- `data_type`：字段类型
- `sample_value`：脱敏样例或样例摘要
- `business_context`：业务场景

训练样本还需要包含：

- `label_id`：人工标注的字段分类标签

## 3. 分类实现

分类模块位于：

`src/data_classification_tool/classifier.py`

核心方法：

- 将字段名、字段注释、数据类型、业务场景、脱敏样例和规则命中特征拼成特征文本；
- 使用 `TfidfVectorizer` 提取字符级 n-gram 特征；
- 使用 `MultinomialNB` 训练贝叶斯分类器；
- 根据标签体系中的关键词和正则模式对贝叶斯概率做加权纠偏；
- 输出预测标签、置信度、候选标签和命中的规则特征。

## 4. 分级实现

分级模块位于：

`src/data_classification_tool/grader.py`

当前默认使用 `RuleBasedGrader`：

- 根据标签体系中的默认等级给出基础等级；
- 如果是敏感个人信息，提升到 L4；
- 如果是一般个人信息，至少 L3；
- 如果有重要数据线索，至少 L3；
- 如果命中密钥、身份证等高敏感模式，提升到 L4；
- 如果分类置信度低于阈值，标记为需要人工复核。

## 5. LLM 接口策略

当前没有 API key，因此默认不调用大模型。

已预留 `LLMGrader`：

- 可以生成结构化 Prompt；
- 支持 OpenAI-compatible API；
- 未配置 `LLM_API_KEY` 时会提示使用 `--grader rule`。

后续有 key 后，可通过 `.env` 配置：

```text
LLM_API_KEY=your-api-key
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini
```

## 6. 运行命令

训练模型：

```powershell
& "D:\conda\envs\data-classification\python.exe" D:\classification\data-classification-tool\scripts\train_model.py
```

分析示例字段：

```powershell
& "D:\conda\envs\data-classification\python.exe" D:\classification\data-classification-tool\scripts\analyze_demo.py
```

运行测试：

```powershell
& "D:\conda\envs\data-classification\python.exe" -m pytest D:\classification\data-classification-tool\tests
```

评估模型：

```powershell
& "D:\conda\envs\data-classification\python.exe" D:\classification\data-classification-tool\scripts\evaluate_model.py
```

启动页面：

```powershell
& "D:\conda\envs\data-classification\python.exe" D:\classification\data-classification-tool\scripts\run_gradio.py
```

评估结果会生成：

- `reports/evaluation/evaluation_metrics.json`：准确率、宏平均 F1、加权 F1、每类 precision/recall/F1
- `reports/evaluation/evaluation_predictions.csv`：每个测试字段的真实标签、预测标签、置信度和候选标签
- `reports/evaluation/confusion_matrix.csv`：混淆矩阵
- `reports/evaluation/evaluation_report.xlsx`：便于人工查看的 Excel 汇总

## 7. 当前阶段判断

当前项目处于“本地可运行 MVP + 初步评估闭环”阶段。

还没有接入真实大模型，也没有接入真实数据库。这个阶段的重点是先把字段样本、标签体系、贝叶斯分类和规则分级跑通，并能用评估报告判断样本质量和模型效果。
